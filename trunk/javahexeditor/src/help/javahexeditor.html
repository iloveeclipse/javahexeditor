javahexeditor User Guide


Introduction

javahexeditor is a java hex editor . It is released as a standalone application using SWT widgets and as Eclipse plugin. It has support for very large files; hex, Ascii and Unicode find; overwrite/insert modes; binary and text cut/copy/paste; undo/redo.


Installation Notes

There are three different possible ways to run javahexeditor :
1- Precompiled in a 32-bit Windows operating system.
2- Standalone java application.
3- Plugin for Eclipse.


Requirements for javahexeditor precompiled:
- 32-bit Windows operating system.

Installation of javahexeditor precompiled:
- Not needed, just double-click on javahexeditor-win32_x.y.z.exe


Requirements for javahexeditor standalone:
- Java 1.4 or later ( java.sun.com ).
- SWT 3.1 or 3.4 ( www.eclipse.org/swt ). Versions 3.2 or 3.3 might work as well.

Installation of javahexeditor standalone:
- Create a new empty directory (folder).
- Unjar (unzip) the net.sourceforge.javahexeditor_x.y.z.jar file contents into the directory.
- Unzip the SWT zip file ( swt-3.x-win32-win32-x86.zip in Windows, swt-3.x-gtk-linux-x86.zip in Linux) into the same directory.
- Windows users: Double-click on gowin32.bat
  Linux users: open a terminal (console), cd to the directory, make golinux runnable ( chmod +x golinux ), and run it ( ./golinux )


Requirements for javahexeditor Eclipse plugin:
- Eclipse 3.1 or 3.4 ( www.eclipse.org ). Versions 3.2 or 3.3 might work as well.

Installation of javahexeditor Eclipse plugin:
- Copy the net.sourceforge.javahexeditor_x.y.z.jar file into the plugins directory in your Eclipse installation.
- Restart Eclipse.
- Any file currently in the workspace (but not part of a jar file) and with one of the extensions listed below can be opened with javahexeditor by right clicking on the file name, expanding the 'Open With' menu option, and selecting javahexeditor .
- File extensions defined for use by javahexeditor : a, bin, cab, class, com, dat, data, drv, dll, exe, hex, iso, jar, lib, o, out, part, pdf, png, ram, res, rom, sys, txt, zip .
- Other file extensions can be added in the following preferences dialog: Window -> Preferences... -> General (+) Editors (+) File Associations


Open files

Use the File -> Open File... menu option.

Drag the file you want to open and drop it inside the editor area.

To start editing a new file from scratch use the File -> New menu option.

Specify the file to edit as a command parameter: javahexeditor reads just the first parameter and interprets it as the file to open. One way this would be done is:
(linux): java -cp .:swt.jar -Djava.library.path=. net.sourceforge.javahexeditor.Manager file.bin
(win32): java -cp .;swt.jar -Djava.library.path=. net.sourceforge.javahexeditor.Manager file.bin


Browsing

You can move around the file as you would in any other text editor. To change from hex to text, or vice versa, press the [Tab] key, or click the mouse in the appropriate area.

To locate a specific position within the file use the Goto dialog (Edit -> Go to Location... or [Ctrl] L). This is especially useful when editing very big files and the scrollbar is not accurate enough.

To select text move the cursor to the selection start and then press the [Shift] key when changing the cursor to the selection end.


Modifying

There are two modify modes: insert and overwrite. You can switch between insert/overwite modes with the [Insert] key in your keyboard (as in some text editors, for instance).

You can insert/delete/trim bytes in insert mode only. In overwrite mode the file length never changes and you cannot paste things bigger than the space left in the file.

You can insert another file(s) by copying it (in explorer, for instance, select the file and then Edit -> Copy), and pasting it (in javahexeditor, Edit -> Paste).

Copying to a text editor has a limit of 4MBytes. Binary copying (from javahexeditor to javahexeditor ) is limited by disk space only.

Modified text is displayed in blue, so it is easier to see the changes made to the file.
